# SW User Story Creator

## Overview

Reads `requirements/vision.md` (generated by `sw-idea-analyzer`) and produces a complete, professional user story document at `requirements/user-stories.md`. Outputs the full Markdown to the console for copy-paste and writes the file simultaneously.

**Language detection:** Read the user's input language and respond entirely in that language. Default to German if undetectable.

---

## Required Input – Check in Order

1. **`requirements/vision.md` exists** → read it, extract idea title, target audience, core features, and risks. Use as the primary source.
2. **`requirements/vision.md` missing, user provided description** → use the description as input.
3. **`requirements/vision.md` missing, no input given** → stop and respond:

   > `requirements/vision.md` not found.
   > Run `/sw-idea-analyzer` first to generate it, or provide your idea directly:
   > `/sw-user-story-creator <your description>`

---

## User Story Format

Every story follows this exact structure:

```
### US-XXX: <Short Title>

**As a** [role], **I want** [goal] **so that** [benefit].

**Priority:** MVP | Nice-to-have

**Acceptance Criteria:**
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3
```

**Rules:**
- Role = specific user type from the target audience (not "user" generically)
- Goal = concrete action, not a vague wish
- Benefit = measurable or observable outcome
- Each story gets 3–5 acceptance criteria minimum
- MVP = required for the first working version; Nice-to-have = can be deferred

---

## Epics

Group stories into epics derived from the core feature areas in the input. Name each epic after the feature cluster it represents.

**Example epics for a task manager:**
- Epic: Authentication & Onboarding
- Epic: Task Management
- Epic: Notifications
- Epic: Settings & Profile

Generate at least one story per core MVP feature. Add 1–2 Nice-to-have stories per epic.

---

## Output Rules

- Show the full `requirements/user-stories.md` content in the console (for copy-paste)
- Also write it to `requirements/user-stories.md`
- Use the user's language throughout
- Never truncate – include every story, every acceptance criterion

---

## File Output

After generating all stories, write to `requirements/user-stories.md` in the repository root.

**Steps:**
1. If `requirements/` does not exist: `mkdir -p requirements`
2. Write the complete document to `requirements/user-stories.md`
3. Show the full Markdown in the console
4. End with:

> ✅ File created: `requirements/user-stories.md`
>
> ▶ Next: Run `/sw-diagram-creator` to visualize the architecture, or start building.
